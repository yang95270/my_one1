get post区别

1.get不安全（数据存储在地址栏） post安全（在请求文本中）

2.get可携带数据小 post大

3.get的效率比post高（数据库传输问题， get不需要往数据库传输）

get:向指定的资源发出“显示”请求。使用GET方法应该只用在读取数据，而不应当被用于产生副作用”的操作中，例如在Web Application中。 其中一个原因是GET可能会被网络蜘蛛等随意访问.

post:向指定资源提交数据，请求服务器进行处理(例如提交表单或者上传文件) .数据被包含在请求本文中。这个请求可能会创建新的资源或修改现有资源，或二者皆有。

服务器状态码

100正在处理 200响应成功 303重定向状态码，需进行附加操作 404服务器资源找不到，通常是路径错误 405禁用请求中指定的方法 500内部错误，通常是Java代码错误 空白（连接失败）服务器没启动

1、地址栏的地址会发生改变

2、不能携带参数

3、请求至少有两次 Response 对象是将服务器端的数据发送到客户端浏览器。

请求转发与重定向的区别/forward和sendRedirect的区别

1.sendRedrect()重定向：不仅可以重定向到同一web应用中的组件，可以重定向到其他站点资源，重定向后浏览器的URL会发生变化，因为重定向的实质 是 server端发送一个消息给浏览器端，使重新浏览新的URL 2.forward() 请求转发：forward只能请求转发给同一web应用中的组件,转发后浏览器的url不会发生改变，forward实质是 站内转发，获取其他server资源，不会给浏览器端发送消息

什么是序列化

序列化就是一种用来处理对象流的机制，所谓对象流也就是将对象的内容进行流化（IO）。我们可以对流化后的对象进行读写操作，也可将流化后的对象传输于网络之间(注：要想将对象传输于网络必须进行流化)！ 反序列化： 把字节序列恢复为Java对象的过程。

为什么要序列化

在对对象流进行读写操作时会引发一些问题，而序列化机制正是用来解决这些问题的！比如 要存储两个类A和B，先实例化{A a = new A(); B b = new B(); }B类包含A的一些方法，如果没有序列化，那么在写入磁盘时A就会多一份 序列化解决的就是给所有对象一个序列号（如1.2.3），保存对象时先检查是否已经存在，如存在则只需写入"与已经保存的具有序列号x的对象相同"的标记，否则，保存该对象

线程池

如果并发的线程数量很多，并且每个线程都是执行一个时间很短的任务就结束了，这样频繁创建线程就会大大降低系统的效率，因为频繁创建线程和销毁线程需要时间。 线程池执行完一个任务，并不被销毁，而是可以继续执行其他的任务
